# Performance refactor — 2026-05-22

Two-pass optimization on the eval path, validated against Apple Pkl
0.31.1 native binary on a 1643-line typed schema (`pkspec/pkl/Test.pkl`).

## Tooling

- `scripts/bench-vs-pkl.sh` — hyperfine-based end-to-end comparison
  between `mpkl eval` (native AOT) and `pkl eval` (native AOT). Same
  fixtures, same shell-less invocation, fairness-guarded by a
  pre-bench `diff` pass that refuses to time fixtures whose outputs
  diverge.
- `moon bench --target js` — per-stage breakdown (tokenize / parse /
  typecheck / eval) for the heavy fixtures, used to localise regressions
  and pinpoint hot leaves.
- `sample` (macOS) on the mpkl release binary — exact stack traces
  from inside a hot eval, used to find the actual leaf doing work.

## Findings

### 1. Stack overflow on `apple-pkl/stdlib/base.pkl`

`synthesize_default_for_type` → `apply_class_defaults_for_type` →
`eval_class_default_members` recursed without threading the cycle
guard. Mutually-recursive class types (`class A { b: B }`,
`class B { a: A }`) blew the JS engine's stack instead of returning
the empty-default shape Apple Pkl produces.

Fix: added `_seen` variants that thread the in-progress class chain
through the recursion; the existing `eval_class_default_members_seen`
already had the guard, but the synthesise / apply pair was resetting
it.

### 2. Reflect source scans were O(L × N) per module

`reflect_module_metadata_value` walks every class and every member;
for each (class, member, kind) it called
`reflect_class_member_decl_line_from_source` and
`doc_comment_before_class_member`, each of which split the module
source into lines and walked them looking for one match. With
`pkspec/Test.pkl` (1643 lines, 49 classes, ~500 reflect queries),
the leaves spent the entire eval budget in `line_declares_reflect_name`
→ `boyer_moore_horspool_find` (confirmed by `sample`).

Fix: introduced `ReflectSourceIndex` — a per-source cache that
pre-splits lines and builds `(class, member, kind) → line index` and
`(name, kind) → line index` maps in a single pass over the source.
Cached at top level keyed by source identity (LRU cap = 4) so repeat
calls within the same eval hit O(1). Doc-comment scanners walk
backwards from the cached line index instead of scanning the full
source each time.

The cycle-guard fix is correctness; the reflect-index fix is the
performance.

## Numbers

### `moon bench --target js`, eval/pkspec-test-schema

| stage | before | after cycle fix | after reflect index | after doc-comment index |
| --- | ---: | ---: | ---: | ---: |
| eval/pkspec-test-schema | 3.43 s | 3.13 s | 1.34 s | **9.58 ms** |
| eval/apple-pkl-stdlib-base | stack overflow | 15.28 s | 12.98 s | 9.80 s |

`eval/pkspec-test-schema` is now within 5x of `parse/pkspec-test-schema`
(1.89 ms) — previously it was ~1700× the parse cost.

### `scripts/bench-vs-pkl.sh`, native AOT, `pkspec/pkl/Test.pkl`

| stage | mpkl mean | pkl mean | ratio (mpkl/pkl) |
| --- | ---: | ---: | ---: |
| baseline (pre-work) | 830 ms (single `time` run, native) | 100 ms | ~8× |
| after cycle fix | not separately measured | | |
| after reflect index | 369 ms | 11 ms | 33× |
| after doc-comment index | **27.2 ms** | 10.2 ms | **2.67×** |

End-to-end on `pkspec/Test.pkl`, mpkl went from ~30× slower than
Apple Pkl to ~2.7× slower — a ~30× reduction in the gap.

## What's left

- `eval/apple-pkl-stdlib-base` still takes ~10 s under JS. That's
  evaluating the entire `pkl:base` stdlib as a user module, which
  isn't a real workload — the stdlib is normally consumed, not
  evaluated. Lower priority than user-facing fixtures.
- `basic-as` / `basic-is` / `basic-rawstring` sit at ~1.25× pkl —
  same family of typed-collection retention / regex / multi-line
  string handling. Profiling would point at the next leaf, but the
  size of the remaining gap doesn't justify another invasive refactor
  yet.
- The reflect-index cache is keyed by source content equality. For
  long-running CLI sessions evaluating many distinct modules, the
  current 4-entry LRU cap may need tuning if a regression appears.
